# Repository Guidelines

## MUST READ

Work from fundamental principles. 

As best you can, use AI-powered solutions instead of hard-coding non-AI systems / crutches.

In general, and especially for agentic systems, DO NOT add nor rely on fallbacks, guardrails, regex classifiers, crutches. Instead, prefer changes to the agentic framework / prompt.

DO NOT over complicate, simplicity is king.

Prefer large magnitude changes over targeted-singular changes.

Investigate fundamentals, audit code, search for the underlying root causes, think about why it didn't work previously. 

Unless otherwise specified, present me with the plan and wait for my "go ahead" before changing the code. You can explore and perform non-code-changing acitons as much as you want before presenting me with the plan. Only ask for the "go ahead" when you have finished investigating and are about to write code, for permission to go write the code.

Make decisions yourself about implementation details, and present me with the reasoning behind said choices.

Prioritize accuracy and understanding over blind speed.

For all ui changes, you must read Uncodixfy/Uncodixfy.md ; if that file doesnt exist yet, you need to pull https://github.com/cyxzdev/Uncodixfy
